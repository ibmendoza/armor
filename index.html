Build Robust Services

Armor is a Go based microservices stack that is pre-baked for production and performance, which will let you focus on solving problems.

Visit on Github →

Developers First

Generate → configure → build → deploy

Get Armor either with go get or by downloading it from Github: 
$ go get github.com/jondot/armor/...
Now, you can use the armor generator: 
$ armor new fourty_two && cd fourty_two
Created fourty_two
Wrote config/development.yaml
Wrote main.go
Wrote Procfile
Wrote Makefile
$ go build && ./fourty_two

Production Tough

Easy doesn't mean flimsy. Armor integrates production proven components together

  • Crazy Fast

    Performance. Baked in from the start with Gin. Then, every component was carefully placed and benchmarked.

  • Production Proven

    Our knowledge and experience distilled into a framework. Then upgraded with only the best components out there.

  • Open Source

    Although modeled after an internal framework. We are focusing on Armor and are accepting pull requests!

    We've Got You Covered

    12-factor microservices with HTTP, RPC, metrics, middleware, and more

    On Shoulders of Giants

    Our mantra. A delicate balance of quality components, our own glue, and exposing it to you

    HTTP

    Armor uses Gin, a popular and fast Go web microframework

    RPC

    For intra-service communication we use gRPC, Google's internal HTTP2/protobuf RPC mechanism

    Configuration

    We use Viper, spf13's config library supporting overrides by environment, files, and flags

    Logging

    Armor uses Logrus for leveled and fan-out logging to terminal, syslog and more

    Metrics

    Using armon's go-metrics we're fanning out metrics to StatsD and others, with a unified API

    Middleware

    We've included a set of Gin middleware that we think is crucial for production microservices

    Health

    Every service comes with a preconfigured health check, exposing essential information

    Containers

    Armor is container happy. Port, interface and more are auto-loaded from the environment

    Examples

    We think useful examples are part of a product. Find yours in /examples :)
